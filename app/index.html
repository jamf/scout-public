Mobile Device Commands

0 Selected Device(s)


Update Inventory

This command will update the inventory of the device in it's Jamf Pro Server. It will be reflected in scout upon next server update.

Lock Device

This command will lock the device or devices it's sent to and require the user to enter their passcode.

Clear Passcode

This command will clear any passcode set on the device. Useful if the user forgot their passcode and is locked out of their device.

Enable Lost Mode

This command will enable lost mode on device(s). This will lock the device. This is useful if a device has been lost or misplaced.

Disable Lost Mode

This command will disable lost mode on device(s). This will unlock the device or devices.

Erase Device

This command will completly wipe the selected devices. Use with extreme caution, this can not be undone.

Set Device Name

This command can be used to set the device name to a string of your choosing.

Add Jamf Pro Server

A sample request will be made to the JSS to verify that credentials are valid and to get the Org Name and Activation Code. A 'ScoutAdmin' user will also be added to this JPS for emergency server access. This user's password is changed every time Scout pulls inventory, and is randomly set.

It is recommended basic inventory is taken roughly once every day, while expanded inventory should be taken on a weekly basis. Limited inventory will perform one API call per Jamf Pro Server, while expanded access will peform a call for every single device, in every single server. It's high recommended these updates are spread throughout the week for more efficent batch processing.

Get Emergency Server Access

If your user has access to get emergency server passwords, you can use the following button to have it returned.
  • This password will be destroyed from the database after being returned in this application one time.
  • The next time this server is scheduled to update, it will update the ScoutAdmin password to a new random password. At that point, this will not longer be valid.
  • If you loose this returned password, you will not be able to get emergency access to this server again until next time it updates.
  • This password is only good for around 24 hours
